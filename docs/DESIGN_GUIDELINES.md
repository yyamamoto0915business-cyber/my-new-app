# MachiGlyph デザインガイドライン

## 核となるコンセプト

> **MachiGlyphは、直感的で迷わないアプリUIを基本にしながら、余白・写真・言葉づかいで地域の温かさと上質さを表現する。**

---

## 実装方針

### 1. 操作性
- アプリらしく分かりやすくする
- **スマホで迷わない導線を最優先**にする
- タップ領域は最低 44px（`--mg-touch-min`）を確保

### 2. 余白
- **余白をしっかり取り、詰め込みすぎない**
- セクション間: `--mg-section-gap`（2rem）
- コンポーネント内: `--mg-space`（1rem）

### 3. 配色
- **やわらかく、清潔感と落ち着き**を出す
- 背景: `--mg-paper`（温かみのある白）
- テキスト: `--mg-ink`（落ち着いた黒）
- 補助テキスト: `--mg-muted`
- アクセント: `--mg-accent`（控えめな真鍮色）
- 境界線: `--mg-line`（薄く控えめ）

### 4. 写真・ビジュアル
- **地域の魅力が丁寧に伝わること**を重視
- アスペクト比を崩さず、適切な余白で表示
- プレースホルダーも温かみのあるトーン

### 5. テキスト
- **短く、親しみやすく、少し上品に**する
- 見出し: `font-serif`（Noto Serif JP 等）
- 本文: `font-body`（M PLUS 1p 等）
- 堅い表現を避け、親しみやすい語尾に

### 6. 画面別の重視点
| 画面 | 重視する価値 |
|------|-------------|
| ホーム | 魅力訴求（地域の良さが伝わる） |
| 主催画面 | 管理しやすさ（効率・ clarity） |
| ストーリー | 読みやすさ（余白、可読性） |

### 7. トーン・バランス
- **おしゃれさよりも、わかりやすさを優先**する
- Otonami のような上質感は取り入れるが、**操作性を損なわない範囲**に留める
- **無機質な管理画面ではなく、毎日見たくなる地域アプリ**を目指す

---

## 禁止・避けること

- 情報を詰め込みすぎた画面
- 小さすぎるタップ領域
- 派手すぎる演出・装飾
- 堅すぎる・事務的な言葉づかい
- 上質感を追求しすぎた複雑な UI

## デザイントークン一覧

| トークン | 役割 |
|---------|------|
| `--mg-paper` | 背景（温かみのある白） |
| `--mg-ink` | メインテキスト |
| `--mg-muted` | 補助テキスト |
| `--mg-line` | 境界線・区切り |
| `--mg-accent` | アクセント・CTA |
| `--mg-accent-soft` | アクセント背景（ホバー等） |
| `--mg-space` | 基本余白（1rem） |
| `--mg-section-gap` | セクション間余白（2rem） |
| `--mg-touch-min` | タップ領域最小（44px） |
| `--mg-radius` | 角丸 |
| `--mg-shadow` | 控えめなシャドウ |
